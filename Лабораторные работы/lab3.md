Тема: Обработка строк в языке C (замена символов)
=================================================


1\. Постановка задачи
---------------------

Требуется разработать программу, которая:

-   считывает строку длиной до 80 символов;
-   заменяет в ней все символы `'a'` и `'b'` на `'A'` и `'B'`;
-   выводит результат на экран.

* * * * *

2\. Полный код программы
------------------------

#include <stdio.h>
#include <windows.h>

int main() {
SetConsoleOutputCP(CP_UTF8);
SetConsoleCP(CP_UTF8);

    char str[81];
    int i;

    printf("Введите строку (до 80 символов): ");
    fgets(str, 81, stdin);

    for (i = 0; str[i] != '\0'; i++) {
        if (str[i] == 'a') {
            str[i] = 'A';
        }
        else if (str[i] == 'b') {
            str[i] = 'B';
        }
    }

    printf("Полученная строка: %s\n", str);

    return 0;
}

* * * * *

3\. Теоретическая основа
========================

3.1. Представление строк в языке C
----------------------------------

В языке C строка --- это массив символов типа `char`, который обязательно заканчивается специальным символом:

'\0'

Этот символ называется **нулевым терминатором** и обозначает конец строки.

Пример:

hello

в памяти:

'h' 'e' 'l' 'l' 'o' '\0'

* * * * *

3.2. Почему массив объявлен как `char str[81]`
----------------------------------------------

char str[81];

Максимальная длина строки по условию --- 80 символов.

Но к ним добавляется ещё один символ:

'\0'

Следовательно:

80 символов + 1 = 81

### Почему нельзя сделать `char str[80]`

В этом случае при вводе строки длиной 80 символов не останется места для `'\0'`, что приведёт к ошибкам при обработке и выводе строки.

* * * * *

3.3. Почему не используется динамическая память
-----------------------------------------------

Можно было бы использовать `malloc`, но в данной задаче:

-   максимальный размер строки известен заранее;
-   динамическое выделение памяти усложняет программу.

Поэтому статический массив - более рациональное решение.

* * * * *

4\. Ввод строки
===============

fgets(str, 81, stdin);

4.1. Назначение функции `fgets`
-------------------------------

Функция `fgets()` считывает строку из потока ввода.

Синтаксис:

fgets(массив, размер, поток);

В данной программе:

-   `str` --- массив для хранения строки;
-   `81` --- максимальное количество символов;
-   `stdin` --- стандартный ввод (клавиатура).

* * * * *

4.2. Почему используется `fgets`, а не `scanf`
----------------------------------------------

Альтернативный вариант:

scanf("%s", str);

Но он **не подходит**, потому что:

-   `scanf("%s")` считывает строку только до первого пробела;
-   `fgets()` считывает всю строку, включая пробелы.

Пример:

Ввод: hello world
scanf → hello
fgets → hello world

* * * * *

4.3. Почему размер равен 81
---------------------------

fgets(str, 81, stdin);

Функция `fgets` гарантирует:

-   чтение не более `size - 1` символов;
-   добавление `'\0'` в конец строки.

Если указать 80, то реально можно будет ввести только 79 символов. Поэтому указывается полный размер массива.

* * * * *

5\. Обработка строки
====================

for (i = 0; str[i] != '\0'; i++)

5.1. Принцип работы цикла
-------------------------

Цикл проходит по всем символам строки, начиная с первого:

i = 0

и заканчивая символом конца строки:

'\0'

* * * * *

5.2. Почему используется `str[i] != '\0'`
-----------------------------------------

Альтернатива:

for (i = 0; i < 80; i++)

Почему это хуже:

-   строка может быть короче 80 символов;
-   программа будет проверять лишние элементы массива;
-   возможна работа с «мусорными» значениями.

Правильный способ --- ориентироваться на фактический конец строки.

* * * * *

5.3. Почему выбран цикл `for`
-----------------------------

Альтернатива:

while (str[i] != '\0')

Оба варианта корректны.

Однако `for` предпочтительнее, потому что:

-   в одной строке задаются все параметры цикла;
-   код компактнее и читается проще.

* * * * *

6\. Проверка и замена символов
==============================

if (str[i] == 'a') {
str[i] = 'A';
}
else if (str[i] == 'b') {
str[i] = 'B';
}

6.1. Оператор сравнения `==`
----------------------------

str[i] == 'a'

означает проверку:

равен ли текущий символ 'a'

Важно отличать:

-   `==` - сравнение;
-   `=` - присваивание.

* * * * *

6.2. Почему символы записываются в одинарных кавычках
-----------------------------------------------------

'a' - это одиночный символ.

"a" - это строка.

В массиве `str` хранятся отдельные символы, поэтому используется `'a'`.

* * * * *

6.3. Почему используется `else if`
----------------------------------

Альтернатива:

if (...) { }
if (...) { }

В данной задаче это допустимо, но менее логично.

Символ не может одновременно быть `'a'` и `'b'`, поэтому:

else if

-   исключает лишнюю проверку;
-   делает код логически последовательным.

* * * * *

6.4. Почему замена выполняется «на месте»
-----------------------------------------

str[i] = 'A';

Это означает, что изменяется исходный массив.

Альтернатива:

-   создать второй массив;
-   записывать туда результат.

Почему это хуже:

-   требует дополнительной памяти;
-   усложняет код;
-   не требуется по условию задачи.

* * * * *

6.5. Почему не используется `toupper()`
---------------------------------------

Функция:

toupper()

переводит символ в верхний регистр.

Но она работает для **всех букв**, а не только для `a` и `b`.

Задача требует заменить только два конкретных символа, поэтому используется ручная проверка.

* * * * *

7\. Вывод результата
====================

printf("Полученная строка: %s\n", str);

7.1. Формат `%s`
----------------

Используется для вывода строки.

Пример:

printf("%s", str);

выведет содержимое массива `str`.

* * * * *

7.2. Почему добавлен `\n`
-------------------------

\n

--- это переход на новую строку.

Без него вывод будет «прилипать» к следующему тексту.

* * * * *

8\. Настройка кодировки
=======================

SetConsoleOutputCP(CP_UTF8);
SetConsoleCP(CP_UTF8);

Назначение
----------

Обеспечивают корректную работу с русским текстом в консоли Windows.

* * * * *

Почему это сделано
------------------

В программе используется русский текст:

"Введите строку..."

Без настройки кодировки возможны проблемы с отображением.

* * * * *

Ограничение
-----------

Этот код работает только в Windows.

На других системах потребуется другой подход.

* * * * *

9\. Завершение программы
========================

return 0;

Возвращает системе код успешного завершения.

* * * * *

10\. Алгоритм работы программы
==============================

1.  Объявляется массив символов.
2.  Пользователь вводит строку.
3.  Программа проходит по строке:
    -   если символ `'a'`, заменяет на `'A'`;
    -   если символ `'b'`, заменяет на `'B'`.
4.  Выводится изменённая строка.

* * * * *

11\. Вывод
==========

Программа демонстрирует базовые принципы работы со строками в языке C:

-   представление строки как массива символов;
-   использование `'\0'` для определения конца строки;
-   безопасный ввод через `fgets`;
-   посимвольную обработку строки;
-   изменение элементов массива по индексу.

Выбранные решения (массив фиксированного размера, цикл до `'\0'`, `fgets`, `if/else if`) являются оптимальными для данной задачи, так как обеспечивают корректность работы, простоту реализации и соответствие условию.
