Тема: Функции с переменным количеством аргументов
=================================================

==================================================

1\. Понятие функции в языке C/C++
---------------------------------

* * * * *

Функция является основным структурным элементом программы.\
Она представляет собой именованный блок кода, выполняющий определённую задачу.

Общий вид функции:

-   возвращаемый тип
-   имя функции
-   список параметров
-   тело функции

Функция может:

-   принимать параметры
-   возвращать результат
-   использовать локальные переменные

Основное назначение функций:

-   разбиение программы на логические части
-   повторное использование кода
-   повышение читаемости программы

* * * * *

2\. Параметры функции
---------------------

* * * * *

Параметры функции --- это данные, которые передаются в неё при вызове.

Существует два типа параметров:

-   обязательные
-   дополнительные

Параметры могут передаваться:

-   по значению
-   по указателю
-   по ссылке

Передача параметров позволяет функции работать с различными данными,\
не изменяя её структуру.

* * * * *

3\. Функции с переменным числом аргументов
------------------------------------------

* * * * *

В языке C/C++ существует возможность создавать функции,\
которые принимают **переменное количество аргументов**.

Синтаксис:

тип имя(тип обязательный_параметр, ...)

Многоточие означает:\
после обязательных параметров можно передать любое количество дополнительных значений

Пример из стандартной библиотеки:

-   ```функция `printf()````

Особенность:

-   компилятор не знает заранее количество аргументов
-   программист обязан сам контролировать их обработку

* * * * *

4\. Обработка переменного числа аргументов
------------------------------------------

* * * * *

Так как количество аргументов заранее неизвестно,\
необходимо передавать дополнительную информацию:

обычно передаётся **количество аргументов**

Пример логики:

-   первый параметр --- количество элементов
-   далее идут сами значения

Это необходимо, потому что:\
язык не хранит автоматически количество переданных параметров

* * * * *

5\. Массивы как модель набора данных
------------------------------------

* * * * *

При работе с переменным числом аргументов логически мы имеем дело\
с набором однотипных элементов.

Такой набор можно рассматривать как:

массив

Массив --- это:

-   структура данных
-   содержащая элементы одного типа
-   расположенные последовательно в памяти

Это позволяет:

-   применять циклы
-   выполнять обработку данных
-   реализовывать алгоритмы

* * * * *

6\. Основные операции над данными
---------------------------------

* * * * *

При обработке набора чисел выполняются базовые операции:

### 6.1 Сумма элементов

Суммирование --- это последовательное сложение всех элементов.

Используется:

-   цикл
-   аккумулятор (переменная для хранения результата)

* * * * *

### 6.2 Поиск максимума

Алгоритм:

1.  принять первый элемент за максимум
2.  пройти по всем элементам
3.  сравнивать
4.  обновлять максимум

* * * * *

### 6.3 Поиск минимума

Аналогичен поиску максимума:

-   только используется противоположное сравнение

* * * * *

### 6.4 Среднее арифметическое

Формула: сумма всех элементов / их количество

Особенность:

-   требуется знать количество элементов

* * * * *

7\. Использование циклов
------------------------

* * * * *

Все операции над набором данных выполняются с помощью циклов.

Наиболее часто используется цикл:

```for```

Назначение:

-   последовательный обход элементов
-   выполнение одинаковых операций

* * * * *

8\. Типы данных
---------------

* * * * *

Для работы с числами используются:

-   ```int``` --- целые числа
-   ```double``` --- вещественные числа

Тип double применяется:

-   при работе с дробными значениями
-   при вычислении среднего

* * * * *

9\. Организация программы в модулях
-----------------------------------

* * * * *

Программа может состоять из нескольких файлов (модулей):

-   основной файл (main)
-   отдельный модуль с функциями
-   заголовочный файл (.h)

Заголовочный файл содержит:

-   прототипы функций

Это необходимо для:

-   раздельной компиляции
-   структурирования программы

* * * * *

10\. Прототипы функций
----------------------

* * * * *

Прототип --- это объявление функции без тела.

Назначение:

-   информировать компилятор о функции
-   обеспечить корректные вызовы

Пример:

```double sum(int count, ...);```

* * * * *

11\. Общая логика решения задачи
--------------------------------

* * * * *

Задача решается следующим образом:

1.  создаются функции:
    -   сумма
    -   максимум
    -   минимум
    -   среднее
2.  каждая функция:
    -   принимает количество аргументов
    -   обрабатывает набор значений
3.  используется цикл для обработки
4.  результат возвращается через return
