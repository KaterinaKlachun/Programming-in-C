Пояснение работы программы\
==========================

Программа представляет собой консольный календарь.\
Пользователь вводит одну строку, а программа сама определяет, что именно нужно сделать:

- если введено `гггг.мм.дд` --- определить день недели\
- если введено `гггг.мм` --- вывести календарь за месяц\
- если введено `гггг` --- вывести календарь за год\
- если введено `now` --- вывести текущую дату

Такой подход выбран потому, что пользовательский ввод может иметь разный формат.\
Поэтому программа сначала работает со строкой, анализирует её, а уже потом преобразует данные в числа.

В лекциях указывается, что строки в C являются нуль-терминальными, то есть хранятся как массив символов до символа `\0`, а для работы со строками используются функции ввода, вывода и обработки строк. :contentReference[oaicite:0]{index=0}

* * * * *

1\. Почему ввод сделан строкой\
-----------------------------

В программе ввод выполняется одной строкой:

```c\
fgets(input, 50, stdin);
```
Это сделано потому, что пользователь может ввести не только числа, но и команду:

```now```

Если бы программа сразу считывала только числа через `scanf`, она не смогла бы корректно обработать разные варианты ввода.

Например:

```2026\
2026.04\
2026.04.26\
now
```
Все эти варианты имеют разную структуру.\
Поэтому сначала программа принимает ввод как текст, а потом уже решает, как его обработать.

Это логично для данной задачи: сначала нужно определить формат, а потом выполнять действие.

* * * * *

1.  Зачем удаляется символ переноса строки

* * * * *

После `fgets` в строке может остаться символ переноса строки `\n`.

Поэтому используется:

```
input[strcspn(input, "\n")] = '\0';
```
Это нужно, чтобы строка была чистой.

Например, пользователь вводит:
```
now
```
Но фактически после `fgets` может получиться:
```
now
```
Если не удалить `\n`, сравнение:
```
strcmp(input, "now")
```
может не сработать, потому что `"now"` и `"now\n"` --- разные строки.

Поэтому символ переноса строки заменяется на `\0`, то есть на конец строки.

* * * * *

1.  Почему команда now проверяется отдельно

* * * * *

Команда:
```
if (strcmp(input, "now") == 0)
```
проверяется до разбора даты.

Это сделано потому, что `now` не является датой и не содержит чисел.

Если пользователь ввёл `now`, программе не нужно:

-   считать точки
-   извлекать год
-   проверять месяц
-   проверять день

Она сразу вызывает:
```
printNow();
```
После этого программа завершается:
```
return 0;
```
Такой порядок делает алгоритм проще: особый случай обрабатывается сразу, а все остальные варианты считаются датами.

* * * * *

1.  Как программа определяет формат даты

* * * * *

Для определения формата используется количество точек в строке.

Функция:
```
countDots(input)
```
проходит по строке и считает символы `.`.

Логика такая:
```
0 точек → введён только год\
1 точка → введён год и месяц\
2 точки → введены год, месяц и день
```
Примеры:
```
2026        → 0 точек\
2026.04     → 1 точка\
2026.04.26  → 2 точки
```
Почему выбран именно такой способ:

-   он простой
-   он подходит под формат задания
-   он позволяет быстро определить, какие данные ввёл пользователь

Это лучше, чем сразу пытаться читать три числа, потому что пользователь может ввести только год или только год с месяцем.

* * * * *

1.  Как работает countDots

* * * * *

Функция:
```
int countDots(char str[])
```
получает строку и проходит по каждому символу до конца строки.

Конец строки определяется символом:
```
'\0'
```
Это стандартный способ обхода строк в C.

Если текущий символ равен точке:
```
if (str[i] == '.')
```
счётчик увеличивается.

В итоге функция возвращает количество точек.

Здесь используются темы:

-   строки как массив символов
-   индексация массива
-   цикл `for`
-   условный оператор `if`

В лекциях указано, что массивы имеют индексы, начинающиеся с 0, а цикл `for` используется для повторения действий с управляющей переменной-счётчиком.

* * * * *

1.  Как происходит разбор года, месяца и дня

* * * * *

После определения количества точек используется `sscanf`.

Например, если точек 0:
```
sscanf(input, "%d", &year)
```
Если точек 1:
```
sscanf(input, "%d.%d", &year, &month)
```
Если точек 2:
```
sscanf(input, "%d.%d.%d", &year, &month, &day)
```
`sscanf` работает как `scanf`, но читает данные не с клавиатуры, а из строки.

Это удобно, потому что:

-   строка уже введена
-   формат уже определён
-   теперь можно извлечь из неё числа

Почему проверяется результат `sscanf`:
```
sscanf(...) == 2
```
или
```
sscanf(...) == 3
```
Потому что функция должна успешно считать нужное количество значений.

Если пользователь ввёл неверный формат, программа не должна продолжать вычисления.

* * * * *

1.  Почему нужна проверка корректности даты

* * * * *

Пользователь может ввести несуществующую дату:
```
2026.13.10\
2026.02.30\
2025.02.29
```
Поэтому есть функция:
```
isValidDate(year, month, day)
```
Она проверяет:

-   год больше 0
-   месяц от 1 до 12
-   день больше 0
-   день не больше количества дней в указанном месяце

Если дату не проверить, программа может посчитать день недели для даты, которой в календаре не существует.

Это плохая логика, поэтому проверка вынесена в отдельную функцию.

* * * * *

1.  Как определяется високосный год

* * * * *

Функция:
```
isLeapYear(int year)
```
определяет, является ли год високосным.

Правило:

год високосный, если он делится на 4 и не делится на 100\
или если он делится на 400

В программе это записано через логические операции:
```
(year % 4 == 0 && year % 100 != 0) || (year % 400 == 0)
```
Здесь используется операция остатка от деления `%`.

Она позволяет проверить, делится ли число нацело.

Например:
```
2024 % 4 == 0 → год делится на 4
```
В лекциях указано, что арифметические операции включают сложение, вычитание, умножение, деление и остаток от деления, причём остаток применяется к целочисленным операндам.

* * * * *

1.  Почему количество дней хранится в массиве

* * * * *

В функции:
```
daysInMonth
```
используется массив:
```
int days[] = {31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};
```
Это сделано потому, что количество дней в месяцах известно заранее.

Массив удобен, потому что:

-   все значения одного типа
-   можно обратиться к нужному месяцу по индексу
-   не нужно писать много `if`

Например:
```
январь → 31\
февраль → 28\
март → 31
```
Но есть важный момент: месяцы считаются от 1 до 12, а индексы массива начинаются с 0.

Поэтому используется:
```
days[month - 1]
```
Если месяц `1`, нужен индекс `0`.\
Если месяц `12`, нужен индекс `11`.

Такое смещение обязательно, иначе программа будет брать неправильное количество дней.

* * * * *

1.  Почему февраль обрабатывается отдельно

* * * * *

В массиве февраль записан как 28 дней:
```
28
```
Но в високосном году февраль имеет 29 дней.

Поэтому в функции есть отдельная проверка:
```
if (month == 2 && isLeapYear(year))
```
Если месяц февраль и год високосный, функция возвращает 29.

Если нет --- возвращает обычное значение из массива.

Такой подход выбран потому, что только февраль зависит от високосности года.\
Остальные месяцы всегда имеют фиксированное количество дней.

* * * * *

1.  Как вычисляется день недели

* * * * *

Функция:
```
dayOfWeek
```
возвращает номер дня недели.

Внутри используется математическая формула.

Результат:

0 → воскресенье\
1 → понедельник\
2 → вторник\
3 → среда\
4 → четверг\
5 → пятница\
6 → суббота

Формула учитывает:

-   год
-   месяц
-   день
-   високосные годы
-   поправки для месяцев

Массив:
```
int t[]
```
содержит поправочные значения для месяцев.

Почему здесь используется формула, а не таблица:

-   таблицу пришлось бы хранить для огромного количества лет
-   формула работает для любой корректной даты
-   программа становится универсальной

* * * * *

1.  Почему январь и февраль считаются частью прошлого года

* * * * *

В функции:
```
if (month < 3) {\
    year--;\
}
```
январь и февраль обрабатываются особым образом.

Это связано с используемой формулой вычисления дня недели.

В таких формулах март часто считается первым месяцем расчётного года, а январь и февраль относятся к предыдущему году.

Это не ошибка, а особенность алгоритма.

Если не сделать это уменьшение года, день недели для январских и февральских дат может считаться неправильно.

* * * * *

1.  Как выводится название дня недели

* * * * *

В функции:
```
printDayOfWeek
```
создаётся массив строк:
```
weekDays[]
```
В нём хранятся названия дней недели.

Функция `dayOfWeek` возвращает число.

Это число используется как индекс массива.

Например:
```
если результат 1 → Понедельник\
если результат 5 → Пятница
```
Такой подход удобнее, чем писать много условий:
```
если 1 → Понедельник\
если 2 → Вторник\
```
...

Массив делает код короче и логичнее.

* * * * *

1.  Как строится календарь месяца

* * * * *

Функция:
```
printMonthCalendar
```
сначала определяет:

-   количество дней в месяце
-   день недели первого числа

Количество дней нужно, чтобы знать, до какого числа выводить календарь.

День недели первого числа нужен, чтобы правильно поставить первое число в таблицу.

Например, если 1 апреля приходится на среду, то перед числом 1 должны быть пустые места за понедельник и вторник.

Затем выводится строка:
```
Пн Вт Ср Чт Пт Сб Вс
```
После этого печатаются пробелы перед первым числом месяца.

Затем цикл выводит все дни от 1 до последнего дня месяца.

* * * * *

1.  Почему в календаре меняется нумерация дней недели

* * * * *

Функция `dayOfWeek` возвращает:
```
0 → воскресенье\
1 → понедельник\
```
...

Но календарь выводится с понедельника:
```
Пн Вт Ср Чт Пт Сб Вс
```
Поэтому результат нужно преобразовать.

Если день недели равен 0, то это воскресенье, а в календарной строке оно должно быть последним.

Поэтому:
```
if (start == 0) {\
    start = 6;\
} else {\
    start--;\
}
```
Что получается:
```
понедельник → 0\
вторник → 1\
...\
воскресенье → 6
```
Это нужно именно для красивого вывода календаря с понедельника.

* * * * *

1.  Как выполняется перенос строки в календаре

* * * * *

При выводе дней используется условие:
```
if ((start + day) % 7 == 0)
```
Оно проверяет, закончилась ли неделя.

Если после вывода очередного дня прошло 7 позиций, выполняется перенос строки.

Здесь снова используется остаток от деления `%`.

Логика такая:

каждые 7 дней нужно переходить на новую строку

Иначе календарь вывелся бы одной длинной строкой.

* * * * *

1.  Как строится календарь года

* * * * *

Функция:
```
printYearCalendar
```
выводит календарь за весь год.

Она не строит год отдельно вручную.

Она просто 12 раз вызывает функцию вывода месяца:
```
for (int month = 1; month <= 12; month++)
```
Это хороший пример повторного использования функции.

Почему так лучше:

-   не нужно писать отдельный код для каждого месяца
-   логика вывода месяца уже готова
-   год --- это просто последовательность 12 месяцев

В лекциях функции рассматриваются как способ разделения программы на логические части и повторного использования кода.

* * * * *

1.  Как работает команда now

* * * * *

Функция:
``
printNow
``
получает текущую дату компьютера.

Для этого используются:
``
time_t\
struct tm\
time()\
localtime()
``
Общая логика:

1.  получить текущее системное время
2.  преобразовать его в локальную дату
3.  извлечь год, месяц и день
4.  вывести день недели для этой даты

Почему к году прибавляется 1900:
```
tm_year
```
хранит количество лет, прошедших с 1900 года.

Например, если сейчас 2026 год, `tm_year` будет 126.

Поэтому:
```
year = tm_year + 1900
```
Почему к месяцу прибавляется 1:
```
tm_mon
```
хранит месяц от 0 до 11.

То есть:
```
0 → январь\
1 → февраль\
...\
11 → декабрь
```
А пользователю нужен обычный номер месяца от 1 до 12.

Поэтому:
```
month = tm_mon + 1
```
* * * * *

1.  Почему программа разбита на функции

* * * * *

В программе много разных задач:

-   проверить високосный год
-   найти количество дней в месяце
-   проверить дату
-   вычислить день недели
-   вывести день недели
-   вывести месяц
-   вывести год
-   вывести текущую дату
-   посчитать точки

Если всё написать внутри `main`, программа станет длинной и трудной для понимания.

Поэтому каждая задача вынесена в отдельную функцию.

Это делает программу:

-   понятнее
-   удобнее для проверки
-   проще для исправления
-   ближе к нормальному стилю программирования

* * * * *

1.  Как работает main

* * * * *

`main` управляет всей программой.

Его логика:

1.  настроить кодировку
2.  создать строку для ввода
3.  запросить дату у пользователя
4.  считать строку
5.  убрать символ переноса строки
6.  проверить `now`
7.  посчитать количество точек
8.  определить формат ввода
9.  извлечь числа
10. проверить корректность
11. вызвать нужную функцию вывода

То есть `main` не выполняет все вычисления сам.

Он только выбирает, какую функцию вызвать.

Это правильно, потому что основная функция должна управлять программой, а не содержать всю логику сразу.

* * * * *

1.  Почему программа сделана именно так

* * * * *

Программа сделана через строковый ввод, потому что формат данных переменный.

Если бы использовался только числовой ввод, было бы сложно отличить:
```
2026
```
от:
```
2026.04
```
и от:
```
2026.04.26
```
Программа сделана через функции, потому что задача состоит из нескольких независимых частей.

Программа использует массивы, потому что:

-   дни месяца удобно хранить списком
-   названия дней недели удобно хранить списком
-   поправки для формулы удобно хранить списком

Программа использует условия, потому что нужно выбирать режим работы.

Программа использует циклы, потому что календарь месяца и года требует повторяющегося вывода.

Такой вариант является нормальным для учебной задачи: он показывает строки, массивы, функции, условия, циклы и работу с датой.Программа является консольным приложением на языке C++. Согласно лекциям, консольное приложение имеет входную точку `main()`, с которой начинается выполнение программы. Для вывода информации на экран используется функция `printf()`, а для ввода данных с клавиатуры --- `scanf()`.

В начале программы подключаются заголовочные файлы:

```
#include <stdio.h>#include <signal.h>#include <conio.h>
```

Файл `stdio.h` нужен для работы с консольным вводом и выводом, то есть для функций `printf()` и `scanf()`. Файл `signal.h` подключается для работы с сигналами. Файл `conio.h` используется для функции `getch()`, которая задерживает закрытие консольного окна после завершения программы.

Далее описана функция:

```
void SignalHandler(int sig)
```

Это пользовательская функция-обработчик. Она принимает один параметр `sig` типа `int`. В этот параметр передаётся номер полученного сигнала. Так как функция ничего не возвращает, у неё указан тип `void`.

Внутри функции используется оператор множественного выбора `switch`. Он проверяет значение переменной `sig` и выбирает нужный вариант обработки. Если был получен сигнал `SIGABRT`, выводится сообщение об аварийном завершении программы. Если `SIGFPE` --- сообщение об ошибке арифметической операции. Если `SIGILL` --- сообщение о недопустимой инструкции. Если `SIGINT` --- сообщение о прерывании с клавиатуры. Если `SIGSEGV` --- сообщение об ошибке доступа к памяти. Если `SIGTERM` --- сообщение о запросе на завершение программы.

После каждого варианта стоит оператор `break`. Он нужен для выхода из блока `switch`, чтобы программа не перешла к следующему варианту.

Функция:

```
void ShowMenu()
```

предназначена для вывода меню на экран. Она также имеет тип `void`, потому что ничего не возвращает. Внутри неё несколько раз вызывается функция `printf()`, с помощью которой пользователю показываются доступные варианты выбора сигнала.

Основная часть программы находится в функции:

```
int main()
```

В ней объявляется переменная:

```
int choice;
```

Эта переменная хранит выбор пользователя. Тип `int` используется потому, что пользователь вводит номер пункта меню.

Далее выполняется регистрация обработчиков сигналов:

```
signal(SIGABRT, SignalHandler);signal(SIGFPE, SignalHandler);signal(SIGILL, SignalHandler);signal(SIGINT, SignalHandler);signal(SIGSEGV, SignalHandler);signal(SIGTERM, SignalHandler);
```

Функция `signal()` связывает конкретный сигнал с функцией-обработчиком. Это значит, что если во время работы программы будет получен один из указанных сигналов, управление будет передано функции `SignalHandler()`.

После этого запускается цикл:

```
do{    ...} while (choice != 0);
```

Используется цикл `do while`, потому что меню должно быть показано хотя бы один раз. Сначала выполняется тело цикла: выводится меню, затем пользователь вводит номер пункта. После этого проверяется условие продолжения цикла. Пока переменная `choice` не равна нулю, программа продолжает работу.

Ввод выбора выполняется командой:

```
scanf("%d", &choice);
```

Спецификатор `%d` означает ввод целого числа. Символ `&` используется для передачи адреса переменной, чтобы введённое значение было записано именно в `choice`.

Затем снова применяется оператор `switch`, но уже для выбора действия по пункту меню. Если пользователь вводит число от 1 до 6, вызывается функция:

```
raise(...)
```

Функция `raise()` искусственно возбуждает выбранный сигнал. Например:

```
raise(SIGSEGV);
```

создаёт сигнал `SIGSEGV`, после чего срабатывает зарегистрированный обработчик `SignalHandler()`.

Если пользователь вводит `0`, программа выводит сообщение о завершении и выходит из цикла. Если введено другое число, выполняется ветка `default`, где выводится сообщение об ошибке выбора.

В конце программы находится:

```
getch();return 0;
```

Функция `getch()` ожидает нажатия клавиши, чтобы окно консоли не закрылось сразу. Оператор `return 0` сообщает операционной системе, что программа завершилась успешно.

Таким образом, программа демонстрирует обработку сигналов. Она регистрирует обработчик для нескольких стандартных сигналов, позволяет пользователю самостоятельно выбрать сигнал, возбуждает его с помощью `raise()` и выводит на консоль сообщение о том, какой сигнал был получен.
