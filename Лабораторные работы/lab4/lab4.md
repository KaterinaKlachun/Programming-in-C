Тема: Модульная программа на C (работа с треугольником)
=======================================================

1\. Постановка задачи
---------------------

Программа должна:

-   считать три стороны треугольника;
-   проверить, существует ли треугольник;
-   если существует:
    -   вычислить периметр;
    -   вычислить площадь;
-   если не существует --- вывести сообщение.

* * * * *

2\. Структура проекта
=====================

Программа разбита на **3 файла**:

main.c        --- точка входа, работа с пользователем\
triangle.c    --- реализация функций\
triangle.h    --- объявления функций (интерфейс)

* * * * *

Почему программа разделена на файлы
-----------------------------------

Это ключевой момент, который часто недопонимают.

### Можно было написать всё в одном файле?

Да. Но это плохая практика.

### Почему разделение --- правильно:

main.c      → отвечает за ввод/вывод (логика программы)\
triangle.c  → содержит вычисления (математика)\
triangle.h  → связывает их между собой

Это даёт:

-   читаемость
-   повторное использование кода
-   разделение ответственности

Если ты это объяснишь на защите --- уже плюс балл.

* * * * *

3\. Файл main.c --- управление программой
=======================================

#include <stdio.h>\
#include <windows.h>\
#include "triangle.h"

Почему подключаются именно эти файлы
------------------------------------

### `stdio.h`

Нужен для:

printf\
scanf

* * * * *

### `windows.h`

SetConsoleOutputCP(CP_UTF8);\
SetConsoleCP(CP_UTF8);

Нужен для корректного русского текста.

Почему не убрать?\
Можно, но тогда русский текст может отображаться криво.

* * * * *

### `"triangle.h"`

Это **ключевой момент**.

#include "triangle.h"

Почему не `triangle.c`?

Потому что:

.h → объявления (что есть)\
.c → реализация (как работает)

Компилятору нужно знать **сигнатуры функций**, а не их код.

* * * * *

4\. Главная функция
===================

int main()

Точка входа в программу.

5\. Переменные
==============

double a;\
double b;\
double c;

Почему используется `double`
----------------------------

Стороны треугольника могут быть:

3.5, 2.7, 10.2

Поэтому нужен тип с дробной частью.

Почему не `int`?

Потому что:

int → только целые\
double → вещественные числа

* * * * *

6\. Ввод данных
===============

scanf("%lf %lf %lf", &a, &b, &c);

Почему `%lf`
------------

%lf → для double\
%d  → для int

Если написать `%d` --- программа сломается.

* * * * *

Почему используется `&`
-----------------------

&a

Это адрес переменной.

`scanf` должен **записать значение**, а не просто прочитать.

* * * * *

7\. Проверка существования треугольника
=======================================

if (a > 0 && b > 0 && c > 0 &&\
a + b > c && a + c > b && b + c > a)

Логика
------

Треугольник существует, если:

1\. Все стороны > 0\
2\. Сумма любых двух сторон больше третьей

* * * * *

Почему именно такие условия
---------------------------

### Ошибка мышления (типичная):

> «Если числа введены --- значит треугольник есть»

Нет.

Пример:

a = 1\
b = 2\
c = 10

Такого треугольника не существует.

* * * * *

Почему не проверить только одно условие
---------------------------------------

Например:

a + b > c

Недостаточно.

Нужно проверять **все три комбинации**.

* * * * *

8\. Вызов функций
=================

perimeter(a, b, c)\
area(a, b, c)

Почему функции вынесены отдельно
--------------------------------

Можно было написать прямо в `main`:

a + b + c

Но:

1\. Код дублируется\
2\. Нарушается читаемость\
3\. Нет переиспользования

Функции делают код:

короче\
понятнее\
структурированнее

* * * * *

9\. Вывод
=========

printf("Периметр: %.2lf\n", ...)

Почему `%.2lf`
--------------

Означает:

вывести число с 2 знаками после запятой

Пример:

12.3456 → 12.35

10\. Файл triangle.h --- интерфейс
================================

#ifndef TRIANGLE_H\
#define TRIANGLE_H

Что это такое
-------------

Это **защита от повторного подключения**.

Почему это нужно:

Если файл подключится дважды --- будет ошибка компиляции.

* * * * *

Объявления функций
------------------

double perimeter(double a, double b, double c);\
double area(double a, double b, double c);

Это **сигнатуры функций**.

Они говорят:

есть функция perimeter\
она принимает 3 double\
и возвращает double

* * * * *

Почему без тела функции
-----------------------

В `.h` файле не пишут реализацию.

Почему?

Потому что:

.h → описание\
.c → реализация

* * * * *

12\. Функция периметра
======================

double perimeter(double a, double b, double c) {\
return a + b + c;\
}

Почему отдельная функция
------------------------

Да, формула простая:

P = a + b + c

Но вынос в функцию даёт:

повторное использование\
понятное имя\
чистый main

* * * * *

13\. Функция площади
====================

double area(double a, double b, double c) {\
double p = perimeter(a, b, c) / 2;\
return sqrt(p * (p - a) * (p - b) * (p - c));\
}

Формула
-------

Используется формула Герона:

S=p(p-a)(p-b)(p-c)S = \sqrt{p(p-a)(p-b)(p-c)}S=p(p-a)(p-b)(p-c)​

где:

p --- полупериметр

* * * * *

Почему сначала считается `p`
----------------------------

double p = perimeter(...) / 2;

Потому что формула требует именно **полупериметр**, а не периметр.

* * * * *

Почему используется `sqrt`
--------------------------

Площадь вычисляется через корень.

Без `sqrt` ты получишь квадрат площади, а не саму площадь.

* * * * *

Почему вызывается `perimeter()`
-------------------------------

Можно было написать:

(a + b + c) / 2

Но вызов функции:

perimeter(a, b, c)

лучше, потому что:

нет дублирования логики\
код единообразный

* * * * *

=========================================

---------------------------------

это уже уровень "нормального разработчика", а не новичка

Плюсы:

повторное использование\

sqrt(отрицательное число)

### 4\. Использование double

Потому что:

математика ≠ только целые числа

Потому что:

в C нет встроенного sqrt без библиотеки
