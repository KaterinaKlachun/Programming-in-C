Тема: Работа с файлами в языке C
================================


1\. Назначение программы
------------------------

Данная программа работает не с вводом из консоли, а с файлами.

Она открывает файл `input.txt`, считывает из него данные о людях:

Фамилия Имя Отчество Год

Затем программа проверяет год рождения. Если год рождения больше `1980`, запись сохраняется в файл `output.txt`.

Во входном файле находятся такие данные: Иванов 1975, Сидоров 1981, Петров 1990, Воробьянинов 1978. В результате в выходной файл попадают только записи с годом больше 1980: Сидоров 1981 и Петров 1990.

* * * * *

2\. Полный код программы
========================

#include <stdio.h>\
#include <windows.h>

int main() {\
SetConsoleOutputCP(CP_UTF8);\
SetConsoleCP(CP_UTF8);

    FILE *in, *out;

    char surname[50], name[50], patronymic[50];\
    int year;

    in = fopen("input.txt", "r");\
    out = fopen("output.txt", "w");

    if (in == NULL || out == NULL) {\
        printf("Ошибка открытия файла\n");\
        return 1;\
    }

    while (fscanf(in, "%s %s %s %d", surname, name, patronymic, &year) == 4) {\
        if (year > 1980) {\
            fprintf(out, "%s %s %s %d\n", surname, name, patronymic, year);\
        }\
    }

    fclose(in);\
    fclose(out);

    printf("Готово. Результат записан в output.txt\n");

    return 0;\
}

* * * * *

3\. Общая идея программы
========================

Программа решает задачу фильтрации данных.

Фильтрация --- это отбор только тех записей, которые подходят под условие.

В этой программе условие такое:

year > 1980

То есть программа оставляет только людей, у которых год рождения больше 1980.

4\. Подключение библиотек
=========================

#include <stdio.h>\
#include <windows.h>

4.1. Библиотека `stdio.h`
-------------------------

#include <stdio.h>

Эта библиотека нужна для стандартного ввода, вывода и работы с файлами.

В программе из неё используются функции:

printf()\
fopen()\
fscanf()\
fprintf()\
fclose()

Без `stdio.h` компилятор не будет знать, что такое `FILE`, `fopen`, `fscanf` и другие функции.

Почему именно `stdio.h`?\
Потому что работа с файлами в языке C относится к стандартному вводу-выводу.

* * * * *

4.2. Библиотека `windows.h`
---------------------------

#include <windows.h>

Она нужна для настройки кодировки консоли Windows:

SetConsoleOutputCP(CP_UTF8);\
SetConsoleCP(CP_UTF8);

Почему она здесь используется?\
В программе есть русский текст:

"Ошибка открытия файла"\
"Готово. Результат записан в output.txt"

Без настройки кодировки в Windows-консоли русский текст иногда отображается неправильно.

Почему это решение не универсальное?\
Потому что `windows.h` работает только в Windows. На Linux или macOS такой код не скомпилируется. Но если программа выполняется в Windows, это нормальное практическое решение.

5\. Главная функция
===================

int main() {

Функция `main()` --- это точка входа в программу.

Именно с неё начинается выполнение.

Тип `int` означает, что функция возвращает целое число.

В конце программы используется:

return 0;

Это означает успешное завершение.

Если произошла ошибка, используется:

return 1;

Это означает завершение с ошибкой.

6\. Настройка кодировки
=======================

SetConsoleOutputCP(CP_UTF8);\
SetConsoleCP(CP_UTF8);

Первая строка настраивает вывод текста в консоль.

Вторая строка настраивает ввод с клавиатуры.

В этой программе ввод с клавиатуры фактически не используется, потому что данные берутся из файла. Но строка:

SetConsoleCP(CP_UTF8);

оставлена для единообразия и на случай, если программу позже расширят.

Почему нельзя просто удалить обе строки?\
Можно, но тогда русский текст в консоли Windows может отображаться некорректно.

* * * * *

7\. Объявление файловых указателей
==================================

FILE *in, *out;

`FILE` --- это специальный тип данных для работы с файлами.

Переменные:

in\
out

являются файловыми указателями.

`in` используется для входного файла.

`out` используется для выходного файла.

* * * * *

Почему используются указатели
-----------------------------

В языке C файл не хранится прямо в переменной.

Переменная типа `FILE*` хранит связь с открытым файлом. Через этот указатель программа понимает, откуда читать данные и куда записывать результат.

Почему два указателя, а не один?\
Потому что программа одновременно работает с двумя разными файлами:

input.txt  --- источник данных\
output.txt --- результат

Если использовать один указатель, программа не сможет удобно различать файл для чтения и файл для записи.

* * * * *

8\. Переменные для данных
=========================

char surname[50], name[50], patronymic[50];\
int year;

Здесь создаются переменные для одной записи из файла.

Каждая строка файла имеет структуру:

Фамилия Имя Отчество Год

Поэтому нужны четыре переменные:

surname     --- фамилия\
name        --- имя\
patronymic  --- отчество\
year        --- год рождения

* * * * *

Почему фамилия, имя и отчество --- массивы `char`
-----------------------------------------------

В языке C строка хранится как массив символов.

Поэтому для фамилии используется:

char surname[50];

Для имени:

char name[50];

Для отчества:

char patronymic[50];

Почему размер 50?\
Потому что фамилия, имя и отчество обычно не длиннее 49 символов. Один символ нужен для `'\0'`, который обозначает конец строки.

Почему не `char surname[10]`?\
Потому что некоторые фамилии могут быть длиннее. Например, `Воробьянинов` уже занимает больше 10 символов с учётом кодировки и окончания строки.

Почему не сделать `char surname[500]`?\
Можно, но это избыточно. Для учебной задачи 50 символов --- разумный запас.

* * * * *

Почему год имеет тип `int`
--------------------------

int year;

Год рождения --- это целое число:

1975\
1981\
1990

Поэтому используется тип `int`.

Почему не `double`?\
Потому что год не бывает дробным значением вида `1981.5`.

* * * * *

9\. Открытие входного файла
===========================

in = fopen("input.txt", "r");

Функция `fopen()` открывает файл.

Здесь открывается файл:

input.txt

в режиме:

"r"

Буква `r` означает read, то есть чтение.

* * * * *

Почему используется режим `"r"`
-------------------------------

Файл `input.txt` нужен только для чтения.

Программа не должна изменять исходные данные.

Поэтому выбран режим:

"r"

Почему не `"w"`?\
Режим `"w"` предназначен для записи. Если открыть существующий файл в режиме `"w"`, его содержимое будет стерто. Для `input.txt` это опасно и неправильно.

Почему не `"a"`?\
Режим `"a"` нужен для добавления данных в конец файла. Здесь добавлять ничего не нужно.

* * * * *

10\. Открытие выходного файла
=============================

out = fopen("output.txt", "w");

Здесь открывается файл:

output.txt

в режиме записи.

Режим:

"w"

означает write, то есть запись.

* * * * *

Почему используется режим `"w"`
-------------------------------

Файл `output.txt` должен содержать новый результат работы программы.

Если файл уже существовал, старое содержимое очищается и записывается новый результат.

Почему это правильно?\
Потому что результат должен соответствовать текущему запуску программы, а не смешиваться со старыми данными.

Почему не `"a"`?\
Режим `"a"` добавлял бы новые строки в конец файла. Тогда при каждом запуске программы результат повторялся бы снова и снова. Получилась бы не программа, а ксерокс с нервным тиком.

* * * * *

11\. Проверка открытия файлов
=============================

if (in == NULL || out == NULL) {\
printf("Ошибка открытия файла\n");\
return 1;\
}

После открытия файлов нужно проверить, получилось ли это сделать.

Если файл не открылся, `fopen()` возвращает:

NULL

* * * * *

Почему проверяются оба файла
----------------------------

in == NULL || out == NULL

Знак:

||

означает «или».

Условие читается так:

если входной файл не открылся или выходной файл не открылся

Тогда программа выводит ошибку и завершает работу.

* * * * *

Почему нельзя продолжать программу без проверки
-----------------------------------------------

Если `input.txt` не открылся, читать данные неоткуда.

Если `output.txt` не открылся, записывать результат некуда.

Если не сделать проверку, программа может работать неправильно или аварийно завершиться.

* * * * *

Почему возвращается `return 1`
------------------------------

return 1;

Это код ошибки.

Он показывает, что программа завершилась неуспешно.

Почему не `return 0`?\
Потому что `0` означает успешное выполнение, а здесь произошла ошибка.

* * * * *

12\. Чтение данных из файла
===========================

while (fscanf(in, "%s %s %s %d", surname, name, patronymic, &year) == 4) {

Это основная часть программы.

Она считывает данные из файла построчно, пока в файле есть корректные записи.

* * * * *

Как работает `fscanf`
---------------------

Функция `fscanf()` похожа на `scanf()`, но читает данные не с клавиатуры, а из файла.

Общий смысл:

fscanf(откуда_читать, формат, куда_записывать);

В программе:

fscanf(in, "%s %s %s %d", surname, name, patronymic, &year)

означает:

прочитать из файла in три строки и одно целое число

* * * * *

Что означает формат
-------------------

"%s %s %s %d"

Он говорит программе, какие данные ожидать:

%s --- строка\
%s --- строка\
%s --- строка\
%d --- целое число

То есть программа ждёт строку вида:

Сидоров Николай Андреевич 1981

* * * * *

Почему у строк нет `&`
----------------------

surname, name, patronymic

Перед строковыми массивами не пишется `&`.

Почему?\
Потому что имя массива уже является адресом его первого элемента.

А для обычной переменной `year` нужен адрес:

&year

Поэтому запись такая:

surname, name, patronymic, &year

* * * * *

Почему проверяется `== 4`
-------------------------

while (fscanf(...) == 4)

Функция `fscanf()` возвращает количество успешно считанных элементов.

В одной записи должно быть 4 элемента:

1\. Фамилия\
2\. Имя\
3\. Отчество\
4\. Год

Если считалось ровно 4 элемента, запись корректна, и её можно обрабатывать.

Почему не написать просто:

while (!feof(in))

Так часто делают новички, но это хуже.

`feof()` показывает конец файла только после неудачной попытки чтения. Из-за этого можно случайно обработать последнюю запись повторно или получить ошибку чтения.

Проверка результата `fscanf()` надёжнее, потому что программа обрабатывает только реально считанные данные.

* * * * *

13\. Проверка условия
=====================

if (year > 1980) {

Программа проверяет год рождения.

Если год больше 1980, запись подходит.

* * * * *

Почему используется `>`, а не `>=`
----------------------------------

Условие:

year > 1980

означает строго больше 1980.

То есть человек с годом рождения 1980 не попадёт в результат.

Если бы нужно было включить 1980 год, надо было бы написать:

year >= 1980

Но в данной программе выбран строгий вариант: только после 1980 года.

* * * * *

14\. Запись результата в файл
=============================

fprintf(out, "%s %s %s %d\n", surname, name, patronymic, year);

Функция `fprintf()` записывает данные в файл.

Она похожа на `printf()`, но выводит не на экран, а в указанный файл.

* * * * *

Почему используется `fprintf`, а не `printf`
--------------------------------------------

`printf()` выводит текст в консоль.

А по задаче результат должен попасть в файл:

output.txt

Поэтому используется:

fprintf(out, ...)

* * * * *

Почему в конце стоит `\n`
-------------------------

%d\n

Символ `\n` переводит запись на новую строку.

Если его убрать, все подходящие записи склеятся в одну строку.

Например, было бы плохо:

Сидоров Николай Андреевич 1981Петров Иван Иванович 1990

Поэтому после каждой записи нужен переход на новую строку.

* * * * *

15\. Закрытие файлов
====================

fclose(in);\
fclose(out);

После завершения работы файлы нужно закрыть.

* * * * *

Почему это обязательно
----------------------

Когда файл открыт, операционная система выделяет для него ресурсы.

Если файл не закрыть:

данные могут не полностью записаться\
файл может остаться занятым\
ресурсы будут освобождены неаккуратно

Особенно важно закрывать файл `out`, потому что данные при записи могут временно храниться в буфере.

`fclose(out)` гарантирует, что всё записано в файл.

* * * * *

16\. Сообщение об успешном завершении
=====================================

printf("Готово. Результат записан в output.txt\n");

После обработки программа сообщает пользователю, что работа завершена.

Почему это полезно?\
Пользователь понимает, что программа не зависла и результат нужно искать в файле `output.txt`.

* * * * *

17\. Завершение программы
=========================

return 0;

Возвращает код успешного завершения.

18\. Почему программа написана именно так
=========================================

Эта программа построена правильно для задачи с файлами.

18.1. Почему используются файлы, а не консольный ввод
-----------------------------------------------------

Если данных много, вводить их вручную каждый раз неудобно.

Файл позволяет:

сохранить исходные данные\
запускать программу несколько раз\
получать результат автоматически

Для обработки списков людей файловый ввод удобнее консольного.

* * * * *

18.2. Почему используется `fscanf`
----------------------------------

Данные имеют простую структуру:

Фамилия Имя Отчество Год

Все части разделены пробелами.

Для такого формата `fscanf()` подходит хорошо.

Почему не `fgets()`?\
`fgets()` считывает строку целиком, а потом её пришлось бы дополнительно разбирать. Это полезно для сложных строк, но здесь формат простой, поэтому `fscanf()` короче и понятнее.

* * * * *

18.3. Почему используется `while`
---------------------------------

Количество записей в файле заранее неизвестно.

В файле может быть:

2 записи\
4 записи\
100 записей

Цикл `while` удобен, потому что работает до тех пор, пока чтение успешно.

* * * * *

18.4. Почему результат записывается в отдельный файл
----------------------------------------------------

Исходный файл лучше не изменять.

`input.txt` --- это исходные данные.

`output.txt` --- это результат обработки.

Так безопаснее: если программа сработает неправильно, исходные данные не будут потеряны.

* * * * *

19\. Что было бы хуже
=====================

Вариант 1: читать данные с клавиатуры
-------------------------------------

Это неудобно, потому что каждый запуск пришлось бы вводить все записи заново.

Файл лучше подходит для списков.

* * * * *

Вариант 2: открыть `input.txt` в режиме `"w"`
---------------------------------------------

Это уничтожило бы исходные данные.

Для чтения нужен режим:

"r"

* * * * *

Вариант 3: использовать `while (!feof(in))`
-------------------------------------------

Такой вариант часто приводит к ошибкам, потому что конец файла определяется только после неудачного чтения.

Правильнее проверять результат `fscanf()`:

== 4

* * * * *

Вариант 4: не закрывать файлы
-----------------------------

Формально маленькая программа может завершиться, и система освободит ресурсы сама.

Но это плохая привычка.

Правильно всегда закрывать файлы вручную через:

fclose()

20\. Ограничения программы
==========================

У программы есть несколько учебных упрощений.

20.1. ФИО не может содержать пробелы внутри части
-------------------------------------------------

Например, фамилия из двух слов не будет считана корректно.

Но для обычного формата:

Фамилия Имя Отчество Год

это нормально.

* * * * *

20.2. Размер строк ограничен 50 символами
-----------------------------------------

Если фамилия или имя будут слишком длинными, возможны проблемы.

Более безопасно было бы ограничить ввод так:

"%49s %49s %49s %d"

Это защитило бы массивы от переполнения.

* * * * *

20.3. Нет отдельной проверки корректности года
----------------------------------------------

Например, программа не проверяет, что год находится в разумном диапазоне.

Можно было бы добавить:

if (year > 1900 && year <= 2026)

Но для простой учебной задачи это не обязательно.

22\. Краткий вывод
==================

Данная программа демонстрирует базовую работу с файлами в языке C. Она читает структурированные данные из входного файла, обрабатывает каждую запись по условию и записывает подходящие строки в выходной файл.

Решение построено именно через `fopen`, `fscanf`, `fprintf` и `fclose`, потому что эти функции предназначены для файлового ввода и вывода. Использование отдельного входного и выходного файла делает программу удобной, безопасной и понятной: исходные данные сохраняются, а результат формируется отдельно.
