Тема: Модульная программа на C для операций над матрицами
=========================================================

1\. Назначение программы
------------------------

Программа предназначена для выполнения операций над двумя квадратными матрицами одинакового размера.

Пользователь вводит:

1\. Размер матриц n\
2\. Элементы матрицы A\
3\. Элементы матрицы B\
4\. Операцию: +, - или *

После этого программа выводит результат:

A + B\
A - B\
A * B

Главная особенность программы --- она разделена на несколько файлов:

main.c    --- основной файл программы\
matrix.c  --- файл с реализацией вычислений\
matrix.h  --- заголовочный файл с объявлением функции

Такой подход называется **модульной организацией программы**.

* * * * *

2\. Общая структура проекта
===========================

В проекте используются три файла.

Файл `main.c`
-------------

Отвечает за:

ввод данных\
выделение памяти\
вызов функции вычисления\
вывод результата

То есть `main.c` управляет программой.

Файл `matrix.c`
---------------

Отвечает за математическую часть:

сложение матриц\
вычитание матриц\
умножение матриц

То есть здесь находится логика вычислений.

Файл `matrix.h`
---------------

Связывает `main.c` и `matrix.c`.

В нём находится объявление функции:

double** calculate(double** A, double** B, int n, char op);

* * * * *

3\. Почему программа разделена на несколько файлов
==================================================

Можно было бы написать весь код в одном `main.c`.

Но это было бы хуже.

Когда вся программа находится в одном файле, код быстро становится длинным и неудобным. Сложнее понять, где ввод данных, где вычисления, где вывод результата.

В этой программе логика разделена:

main.c    --- общение с пользователем\
matrix.c  --- вычисления\
matrix.h  --- описание доступных функций

Такой подход удобнее, потому что каждый файл отвечает за свою часть работы.

Это похоже на нормальную организацию проекта: не сваливать всё в одну коробку, а разложить по папкам. Иначе потом не программа, а кладовка после ремонта.

* * * * *

4\. Файл main.c
===============

#include <stdio.h>\
#include <stdlib.h>\
#include <windows.h>\
#include "matrix.h"

4.1. Подключение `stdio.h`
--------------------------

#include <stdio.h>

Эта библиотека нужна для функций ввода и вывода:

printf()\
scanf()

`printf()` используется, чтобы выводить сообщения на экран.

`scanf()` используется, чтобы считывать данные с клавиатуры.

Почему именно `stdio.h`?\
Потому что в языке C стандартный ввод и вывод находятся именно в этой библиотеке.

* * * * *

4.2. Подключение `stdlib.h`
---------------------------

#include <stdlib.h>

Эта библиотека нужна для работы с динамической памятью.

В программе используется функция:

malloc()

Она выделяет память во время выполнения программы.

Почему здесь нужна динамическая память?\
Потому что размер матрицы заранее неизвестен. Пользователь сам вводит `n`.

Например:

n = 2\
n = 3\
n = 10

Программа должна уметь создать матрицу любого введённого размера.

Если бы размер был известен заранее, можно было бы написать:

double A[3][3];

Но здесь размер задаёт пользователь, поэтому используется `malloc()`.

* * * * *

4.3. Подключение `windows.h`
----------------------------

#include <windows.h>

Эта библиотека нужна для настройки кодировки консоли Windows.

В программе есть строки:

SetConsoleOutputCP(CP_UTF8);\
SetConsoleCP(CP_UTF8);

Они нужны, чтобы русский текст в консоли отображался корректно.

Почему это не универсальное решение?\
Потому что `windows.h` работает только в Windows. На Linux или macOS такой код не подойдёт.

Но если программа запускается на Windows, это нормальное практическое решение.

* * * * *

4.4. Подключение собственного файла `matrix.h`
----------------------------------------------

#include "matrix.h"

Здесь подключается заголовочный файл с объявлением функции `calculate`.

Почему используется `"matrix.h"`, а не `<matrix.h>`?

Двойные кавычки используются для собственных файлов проекта:

#include "matrix.h"

Угловые скобки используются для стандартных библиотек:

#include <stdio.h>

Почему не подключается `matrix.c`?\
Потому что `.c` файл содержит реализацию, а `.h` файл содержит объявление. В C принято подключать именно заголовочные файлы.

* * * * *

5\. Главная функция
===================

int main() {

Функция `main()` --- это точка входа в программу.

Именно с неё начинается выполнение.

Тип `int` означает, что программа в конце должна вернуть целое число.

Обычно:

return 0;

означает успешное завершение программы.

* * * * *

6\. Настройка кодировки
=======================

SetConsoleOutputCP(CP_UTF8);\
SetConsoleCP(CP_UTF8);

Первая строка настраивает вывод русского текста.

Вторая строка настраивает ввод текста с клавиатуры.

Почему нужны обе?\
Потому что ввод и вывод --- это разные направления работы с консолью.

Если настроить только вывод, русский текст программы может отображаться нормально, но ввод может работать некорректно.

* * * * *

7\. Объявление переменных
=========================

int n, i, j;\
char op;

Переменная `n`
--------------

int n;

Хранит размер матриц.

Если пользователь вводит:

3

то создаются матрицы размером:

3 x 3

Почему `int`?\
Потому что размер матрицы --- это целое число.

* * * * *

Переменные `i` и `j`
--------------------

int i, j;

Используются как счётчики в циклах.

`i` обычно обозначает номер строки.

`j` обычно обозначает номер столбца.

Для матрицы это удобно:

A[i][j]

где:

i --- строка\
j --- столбец

* * * * *

Переменная `op`
---------------

char op;

Хранит символ операции:

+\
-\
*

Почему тип `char`?\
Потому что операция --- это один символ.

* * * * *

8\. Ввод размера матрицы
========================

printf("Введите размер матриц n: ");\
scanf("%d", &n);

Сначала программа выводит сообщение пользователю.

Затем считывает число `n`.

%d

используется для ввода целого числа.

&n

означает адрес переменной `n`.

Почему нужен `&`?\
Потому что `scanf()` должна записать введённое значение в переменную. Для этого ей нужен адрес, где эта переменная находится.

* * * * *

9\. Выделение памяти под матрицы
================================

double** A = (double**)malloc(n * sizeof(double*));\
double** B = (double**)malloc(n * sizeof(double*));

Здесь создаются две матрицы `A` и `B`.

Тип:

double**

означает указатель на указатель.

Проще говоря, это способ хранить двумерный массив через динамическую память.

* * * * *

Почему используется `double`
----------------------------

Матрицы могут содержать дробные числа:

1.5\
2.25\
3.7

Поэтому используется тип `double`.

Почему не `int`?\
Потому что `int` хранит только целые числа. Если пользователь введёт дробное значение, точность будет потеряна или ввод сработает неправильно.

* * * * *

Почему используется `malloc`
----------------------------

Размер матрицы вводится пользователем во время выполнения программы.

Например, заранее неизвестно, будет матрица:

2x2\
3x3\
5x5

Поэтому нельзя заранее написать фиксированный массив.

Динамическая память позволяет создать матрицу нужного размера уже после ввода `n`.

* * * * *

10\. Выделение памяти под строки матриц
=======================================

for (i = 0; i < n; i++) {\
A[i] = (double*)malloc(n * sizeof(double));\
B[i] = (double*)malloc(n * sizeof(double));\
}

Сначала выделяется память под массив указателей:

double** A

Но этого ещё недостаточно.

Каждая строка матрицы тоже должна получить память.

Цикл проходит по всем строкам и для каждой строки выделяет место под `n` чисел типа `double`.

Если `n = 3`, то создаётся структура:

A[0] -> 3 числа\
A[1] -> 3 числа\
A[2] -> 3 числа

То же самое для матрицы `B`.

* * * * *

Почему память выделяется в два этапа
------------------------------------

Матрица через `double**` устроена как массив строк.

Сначала создаётся список строк, потом каждая строка создаётся отдельно.

Поэтому и нужно два этапа:

1\. Выделить память под указатели на строки\
2\. Выделить память под элементы каждой строки

Почему не использовать обычный массив `double A[n][n]`?\
В современных компиляторах C это иногда возможно, но не всегда удобно и не всегда поддерживается одинаково. Динамическая память через `malloc()` более универсальна для задач, где размер задаётся пользователем.

* * * * *

11\. Ввод матрицы A
===================

printf("Введите матрицу A (%dx%d):\n", n, n);\
for (i = 0; i < n; i++)\
for (j = 0; j < n; j++)\
scanf("%lf", &A[i][j]);

Здесь программа считывает элементы матрицы `A`.

Используются два вложенных цикла.

Внешний цикл отвечает за строки:

for (i = 0; i < n; i++)

Внутренний цикл отвечает за столбцы:

for (j = 0; j < n; j++)

Элемент матрицы записывается так:

A[i][j]

* * * * *

Почему нужны два цикла
----------------------

Матрица --- двумерная структура.

У неё есть:

строки\
столбцы

Один цикл прошёл бы только по одной размерности.

Для матрицы нужно пройти по каждой ячейке:

строка 0, столбец 0\
строка 0, столбец 1\
строка 0, столбец 2\
строка 1, столбец 0\
...

Поэтому используются два цикла.

* * * * *

Почему `%lf`
------------

scanf("%lf", &A[i][j]);

`%lf` используется для ввода значения типа `double`.

Если использовать `%d`, программа ожидала бы целое число типа `int`, а это другой тип данных.

* * * * *

12\. Ввод матрицы B
===================

printf("Введите матрицу B:\n");\
for (i = 0; i < n; i++)\
for (j = 0; j < n; j++)\
scanf("%lf", &B[i][j]);

Матрица `B` вводится таким же способом, как и матрица `A`.

Почему код похожий?\
Потому что обе матрицы имеют одинаковый размер `n x n` и одинаковый тип данных.

* * * * *

13\. Ввод операции
==================

printf("Введите операцию (+, -, *): ");\
scanf(" %c", &op);

Пользователь вводит один символ операции.

Обрати внимание на пробел перед `%c`:

" %c"

Это важно.

* * * * *

Почему перед `%c` стоит пробел
------------------------------

После ввода чисел в буфере может остаться символ перехода на новую строку `\n`.

Если написать:

scanf("%c", &op);

то программа может считать этот `\n` вместо настоящей операции.

Пробел перед `%c` говорит `scanf()` пропустить пробелы, переносы строк и другие пустые символы.

Поэтому правильно:

scanf(" %c", &op);

Это маленькая деталь, но именно такие мелочи потом часами ломают нервы.

* * * * *

14\. Вызов функции вычисления
=============================

double** result = calculate(A, B, n, op);

Здесь вызывается функция `calculate`.

Она получает:

A  --- первая матрица\
B  --- вторая матрица\
n  --- размер матриц\
op --- операция

И возвращает новую матрицу `result`.

Почему вычисления вынесены в отдельную функцию?\
Потому что `main()` не должен превращаться в огромный файл со всей логикой.

`main()` должен управлять программой, а вычисления лучше вынести в отдельный модуль.

* * * * *

15\. Вывод результата
=====================

printf("Результат:\n");\
for (i = 0; i < n; i++) {\
for (j = 0; j < n; j++) {\
printf("%.lf ", result[i][j]);\
}\
printf("\n");\
}

Снова используются два цикла, потому что результат --- это тоже матрица.

printf("%.lf ", result[i][j]);

выводит число без знаков после запятой.

* * * * *

Почему `%.lf`
-------------

Формат:

%.lf

означает вывести число типа `double` без дробной части.

Например:

3.0 → 3\
4.7 → 5

То есть значение округляется при выводе.

Если нужно показывать два знака после запятой, лучше использовать:

%.2lf

В учебной задаче `%.lf` допустимо, если вводятся в основном целые значения. Но если матрицы дробные, лучше выводить точнее.

* * * * *

16\. Файл matrix.h
==================

#ifndef MATRIX_H\
#define MATRIX_H

double** calculate(double** A, double** B, int n, char op);

#endif

Это заголовочный файл.

Он содержит объявление функции `calculate`.

* * * * *

Зачем нужен `matrix.h`
----------------------

Файл `main.c` вызывает функцию `calculate`, но сама реализация находится в `matrix.c`.

Чтобы компилятор понял, что такая функция существует, ему нужно объявление:

double** calculate(double** A, double** B, int n, char op);

* * * * *

Что означает объявление функции
-------------------------------

double** calculate(double** A, double** B, int n, char op);

Эта строка сообщает:

функция называется calculate\
принимает две матрицы double**\
принимает размер int n\
принимает символ операции char op\
возвращает матрицу double**

* * * * *

Почему в `.h` нет тела функции
------------------------------

В заголовочном файле обычно пишут только объявления.

Реальный код функции находится в `.c` файле.

То есть:

matrix.h  --- что существует\
matrix.c  --- как это работает

* * * * *

Зачем нужны `#ifndef`, `#define`, `#endif`
------------------------------------------

#ifndef MATRIX_H\
#define MATRIX_H\
...\
#endif

Это защита от повторного подключения файла.

Если `matrix.h` случайно подключится несколько раз, компилятор не будет повторно читать одно и то же объявление.

Почему это важно?\
В больших проектах заголовочные файлы могут подключаться через разные файлы. Без такой защиты возможны ошибки повторного объявления.

* * * * *

17\. Файл matrix.c
==================

#include <stdlib.h>\
#include "matrix.h"

`stdlib.h` нужен для `malloc()`.

`matrix.h` подключается, чтобы реализация функции соответствовала объявлению.

* * * * *

18\. Функция calculate
======================

double** calculate(double** A, double** B, int n, char op) {

Функция получает две матрицы и операцию, которую нужно выполнить.

Она возвращает новую матрицу с результатом.

* * * * *

Почему функция возвращает `double**`
------------------------------------

Результатом сложения, вычитания или умножения матриц является новая матрица.

Так как матрицы в программе представлены типом `double**`, результат тоже имеет тип:

double**

* * * * *

19\. Локальные переменные функции
=================================

int i, j, k;

`i` --- номер строки.

`j` --- номер столбца.

`k` --- дополнительный счётчик для умножения матриц.

Для сложения и вычитания хватает `i` и `j`.

Для умножения нужен ещё `k`, потому что элемент результата считается через сумму произведений.

* * * * *

20\. Создание матрицы результата
================================

double** result = (double**)malloc(n * sizeof(double*));\
for (i = 0; i < n; i++) {\
result[i] = (double*)malloc(n * sizeof(double));\
}

Функция создаёт новую матрицу `result`.

Почему не записывать результат в `A` или `B`?\
Потому что исходные матрицы лучше не изменять. Если изменить `A`, то исходные данные будут потеряны.

Создание отдельной матрицы результата делает программу логичнее:

A и B --- исходные данные\
result --- результат вычисления

* * * * *

21\. Сложение матриц
====================

if (op == '+') {\
for (i = 0; i < n; i++)\
for (j = 0; j < n; j++)\
result[i][j] = A[i][j] + B[i][j];\
}

Сложение матриц выполняется поэлементно.

То есть каждый элемент результата равен сумме элементов с такими же индексами:

result[i][j] = A[i][j] + B[i][j]

Пример:

A = 1 2\
3 4

B = 5 6\
7 8

A + B = 6  8\
10 12

* * * * *

Почему сложение выполняется именно так
--------------------------------------

Матрицы складываются только по соответствующим элементам.

Нельзя складывать строку с другой строкой целиком как одно число.

Каждая ячейка результата зависит от двух ячеек с тем же положением.

* * * * *

22\. Вычитание матриц
=====================

else if (op == '-') {\
for (i = 0; i < n; i++)\
for (j = 0; j < n; j++)\
result[i][j] = A[i][j] - B[i][j];\
}

Вычитание матриц тоже выполняется поэлементно:

result[i][j] = A[i][j] - B[i][j]

Пример:

A = 5 7\
9 4

B = 1 2\
3 4

A - B = 4 5\
6 0

* * * * *

23\. Умножение матриц
=====================

else if (op == '*') {\
for (i = 0; i < n; i++) {\
for (j = 0; j < n; j++) {\
result[i][j] = 0;\
for (k = 0; k < n; k++) {\
result[i][j] += A[i][k] * B[k][j];\
}\
}\
}\
}

Умножение матриц сложнее, чем сложение и вычитание.

Элемент результата находится как сумма произведений элементов строки первой матрицы и столбца второй матрицы.

Формула:

result[i][j] = A[i][0] * B[0][j] + A[i][1] * B[1][j] + ... + A[i][n-1] * B[n-1][j]

* * * * *

Почему нужны три цикла
----------------------

Для умножения матриц нужно:

i --- выбрать строку матрицы A\
j --- выбрать столбец матрицы B\
k --- пройти по элементам строки и столбца

Поэтому используются три вложенных цикла.

Для каждого элемента `result[i][j]` программа считает сумму.

* * * * *

Почему перед внутренним циклом стоит `result[i][j] = 0`
-------------------------------------------------------

result[i][j] = 0;

Перед суммированием ячейку результата нужно обнулить.

Потом в неё постепенно добавляются произведения:

result[i][j] += A[i][k] * B[k][j];

Если не обнулить значение, в ячейке может быть мусорное значение из памяти. Тогда результат будет неправильным.

* * * * *

24\. Возврат результата
=======================

return result;

Функция возвращает указатель на созданную матрицу результата.

После этого `main.c` получает результат в переменную:

double** result = calculate(A, B, n, op);

* * * * *

25\. Почему операция выбирается через `if / else if`
====================================================

В программе используется:

if (op == '+') {\
...\
}\
else if (op == '-') {\
...\
}\
else if (op == '*') {\
...\
}

Такой способ понятный и прямой.

Можно было бы использовать `switch`:

switch (op)

Для выбора по одному символу `switch` тоже подходит.

Почему здесь `if / else if` нормально?\
Потому что операций всего три, код остаётся читаемым. Для учебной программы это допустимый и понятный вариант.

Если операций станет больше, например деление, транспонирование, определитель, тогда `switch` может выглядеть аккуратнее.

* * * * *

26\. Почему программа работает только с квадратными матрицами
=============================================================

Пользователь вводит только одно число:

n

Значит обе матрицы имеют размер:

n x n

Это упрощает программу.

Почему не вводятся отдельно строки и столбцы?\
Потому что задача ориентирована на квадратные матрицы одинакового размера.

Для сложения и вычитания матрицы должны быть одинакового размера.

Для умножения матриц общее правило сложнее: количество столбцов первой матрицы должно быть равно количеству строк второй.

Но если обе матрицы `n x n`, то умножение всегда возможно.

Поэтому выбор квадратных матриц упрощает проверку условий и делает программу учебной, а не монстром с щупальцами.

* * * * *

27\. Почему выбран тип `double**`, а не обычный двумерный массив
================================================================

Обычный массив фиксированного размера выглядел бы так:

double A[3][3];

Но тогда размер матрицы жёстко задан.

Если пользователь захочет матрицу `4x4`, такой массив уже не подойдёт.

`double**` вместе с `malloc()` позволяет создать матрицу нужного размера во время выполнения программы.

Именно поэтому используется динамическая память.

* * * * *

28\. Важное замечание: в программе есть слабые места
====================================================

Как учебный материал важно не только хвалить код, но и видеть ограничения.

28.1. Нет проверки `n`
----------------------

Сейчас пользователь может ввести:

0\
-3

И программа попытается выделить некорректную память.

Лучше было бы проверить:

if (n <= 0) {\
printf("Размер матрицы должен быть положительным.\n");\
return 1;\
}

* * * * *

28.2. Нет проверки операции
---------------------------

Если пользователь введёт:

/

функция `calculate()` всё равно вернёт матрицу `result`, но её элементы не будут нормально заполнены.

Лучше добавить обработку неверной операции.

* * * * *

28.3. Нет освобождения памяти
-----------------------------

Память выделяется через `malloc()`, но не освобождается через `free()`.

В конце программы желательно освободить:

free(A[i]);\
free(B[i]);\
free(result[i]);\
free(A);\
free(B);\
free(result);

Для маленькой учебной программы это не всегда критично, потому что после завершения программы система сама освободит память. Но с точки зрения правильного C-кода память нужно освобождать вручную.

* * * * *

28.4. Нет проверки результата `malloc`
--------------------------------------

`malloc()` может вернуть `NULL`, если память не выделилась.

В надёжной программе это нужно проверять.

* * * * *

29\. Почему всё равно этот вариант подходит для учебной задачи
==============================================================

Несмотря на ограничения, программа хорошо показывает важные темы языка C:

динамическая память\
двумерные массивы через указатели\
модульная структура программы\
заголовочные файлы\
функции\
операции над матрицами

Для учебной лабораторной работы это хороший уровень: программа не просто считает матрицы, а показывает, как разделять код на части.

30\. Итоговый алгоритм работы программы
=======================================

1\. Подключаются необходимые библиотеки.\
2\. Пользователь вводит размер матриц n.\
3\. Программа выделяет память под матрицы A и B.\
4\. Пользователь вводит элементы матрицы A.\
5\. Пользователь вводит элементы матрицы B.\
6\. Пользователь выбирает операцию: +, - или *.\
7\. Функция calculate() создаёт матрицу результата.\
8\. В зависимости от операции выполняется:\
- сложение;\
- вычитание;\
- умножение.\
  9\. Результат возвращается в main.c.\
  10\. Программа выводит итоговую матрицу на экран.

31\. Краткий вывод
==================

Данная программа демонстрирует работу с квадратными матрицами в языке C. Она использует динамическое выделение памяти, потому что размер матриц заранее неизвестен и задаётся пользователем. Код разделён на несколько файлов: основной файл отвечает за ввод и вывод, файл `matrix.c` --- за вычисления, а `matrix.h` связывает эти части между собой.

Такой способ организации выбран потому, что он делает программу понятнее, аккуратнее и ближе к реальной структуре проектов на C. Вместо того чтобы писать всё в одном `main`, вычисления вынесены в отдельный модуль. Это улучшает читаемость кода и позволяет легче изменять программу в будущем.
