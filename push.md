# Как скачать и запустить проект на другом ПК через CLion

Репозиторий проекта:

`https://github.com/KaterinaKlachun/Programming-in-C.git`

## 1. Что нужно на другом ПК

На другом компьютере должен быть установлен:

- CLion
- Git
- компилятор C/C++
- CMake

Если CLion установлен, но ещё ни разу не использовался, сначала нужно выполнить его первоначальную настройку.

---

## 2. Первичная настройка CLion

После первого запуска CLion нужно проверить, что у него есть инструменты для сборки проекта.

### Открыть настройки:

**File → Settings → Build, Execution, Deployment → Toolchains**

Там должны быть указаны:

- **C Compiler**
- **C++ Compiler**
- **Debugger**
- **CMake**

Если поля пустые, значит CLion ещё не настроен.

### Что делать дальше

#### Вариант 1. Если установлен MinGW

Нужно указать путь к MinGW, например:

```text
C:\MinGW
```

или

```text
C:\msys64\mingw64
```

После этого CLion сам подхватит:
- gcc 
- g++ 
- gdb

#### Вариант 2. Если компилятора нет

Нужно установить один из вариантов:

- MinGW-w64 
- MSYS2 
- Visual Studio Build Tools

Для учебного проекта проще всего использовать MinGW-w64.

## 3. Как скачать проект из GitHub

Есть два способа.

#### Способ 1. Через CLion
Открыть CLion
На стартовом экране выбрать Get from VCS
Вставить ссылку:
```https://github.com/KaterinaKlachun/Programming-in-C.git```

Выбрать папку, куда скачать проект

Нажать Clone

#### Способ 2. Через терминал

Открыть терминал и выполнить:

```git clone https://github.com/KaterinaKlachun/Programming-in-C.git```

cd Programming-in-C

После этого можно открыть папку проекта через CLion.

## 4. Как открыть проект в CLion
- В CLion выбрать Open 
- Указать папку Programming-in-C 
- Дождаться, пока CLion загрузит проект 
- CLion автоматически прочитает файл CMakeLists.txt 

Если toolchain настроен правильно, проект соберётся автоматически.

## 5. Если CLion пишет ошибку при открытии

Если появляются ошибки вида:
- compiler not found 
- CMake is not configured 
- toolchain is not configured

нужно вернуться в:

File → Settings → Build, Execution, Deployment → Toolchains

и проверить, что указаны:
- компилятор 
- debugger 
- CMake

## 6. Как собрать проект

После открытия проекта CLion обычно сам запускает CMake.

Если этого не произошло, нужно:
- Открыть файл CMakeLists.txt 
- Подождать, пока сверху появится предложение загрузить CMake-проект 
- Нажать Load CMake Project

После этого нажать кнопку Run или Build.

## 7. Какой файл запускать

Если в CMakeLists.txt указаны цели:
- lab1
- lab2
- lab3
- lab4

то в CLion можно выбрать нужную конфигурацию запуска и запустить нужную лабораторную работу.

Например, для лабораторной №4 нужно выбрать:

lab4

и нажать Run.

## 8. Почему в репозитории нет папки cmake-build-debug

Папка cmake-build-debug не хранится в GitHub, потому что она создаётся автоматически на каждом компьютере при сборке проекта.

Она содержит:
- временные файлы
- кэш CMake
- объектные файлы
- .exe

Это не исходный код, поэтому такие файлы не должны храниться в репозитории.

## 9. Краткий порядок действий
- Установить и открыть CLion
- Проверить настройку Toolchains
- Скачать проект из GitHub
- Открыть папку проекта
- Дождаться загрузки CMakeLists.txt
- Выбрать нужную конфигурацию запуска
- Нажать Run
