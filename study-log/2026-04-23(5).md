## 1. Структуры в языке C

Структура - это пользовательский тип данных, который позволяет объединить в одном объекте несколько полей разных типов.

В обычной переменной можно хранить только одно значение:
- одно число
- одну строку
- один символ

Но в реальной задаче часто нужно хранить набор связанных характеристик одного объекта.
Например, если описывается человек, то у него есть:
- имя
- фамилия
- год рождения
- пол
- рост

Если хранить эти данные отдельно в разных массивах, программа становится неудобной и легко ошибиться:
- имя одного человека может смешаться с фамилией другого
- данные трудно передавать
- сортировка усложняется

Именно поэтому используется структура.

Пример структуры:
```c\
struct human {
    char name[50];
    char surname[50];
    int year;
    char gender[10];
    double height;
};
```

Что это означает:

-   `char name[50]` - строка для имени
-   `char surname[50]` - строка для фамилии
-   `int year` - год рождения
-   `char gender[10]` - строка для пола
-   `double height` - рост в метрах

Таким образом, структура `human` описывает одного человека как единый объект.

2\. Массивы структур
--------------------

После того как описан тип структуры, можно создавать переменные этого типа.

Пример одной переменной:

```struct human person;```

Но в задаче нужен не один человек, а список людей.
Для этого удобно создать массив структур:

```struct human arr[100];```

Это означает:

-   `arr` - массив
-   каждый элемент массива имеет тип `struct human`
-   всего можно хранить до 100 записей

Массив структур - это набор однотипных объектов, каждый из которых содержит сразу несколько полей.

Это особенно удобно, когда:

-   данные считываются из файла
-   их нужно сортировать
-   их нужно выводить в определённом порядке

3\. Обращение к полям структуры
-------------------------------

Чтобы получить доступ к отдельному полю структуры, используется оператор точки.

Пример:

```
arr[i].year
arr[i].name
arr[i].surname
```

Что здесь происходит:

-   `arr[i]` - это i-я запись
-   `.year` - поле года рождения
-   `.name` - поле имени

Таким образом, запись:

```arr[i].year```

означает:
«взять i-го человека и обратиться к его году рождения».

Это базовый механизм работы со структурами в языке C.

4\. Работа с текстовыми файлами
-------------------------------

В задаче данные хранятся не в программе, а в текстовом файле.

Текстовый файл удобен тем, что:

-   его можно редактировать вручную
-   он хранит записи построчно
-   его легко открыть и прочитать

Пример содержимого файла:

Иванов Петр 1975 м 1.78
Сидоров Николай 1981 м 1.82
Петрова Катя 2005 ж 1.65

Каждая строка содержит одну запись:

-   фамилия
-   имя
-   год рождения
-   пол
-   рост

Для работы с файлами используется тип:

```FILE *f;```

Это файловый указатель, через который программа получает доступ к файлу.

Открытие файла:

```fopen("input.txt", "r")```

Здесь:

-   `"input.txt"` - имя файла
-   `"r"` - режим чтения

Если файл не найден, `fopen` возвращает `NULL`, поэтому нужна проверка:

```if (f == NULL)```

5\. Чтение данных из файла
--------------------------

Так как каждая строка файла имеет одинаковый формат, удобно считывать записи сразу по шаблону:

```
fscanf(f, "%49s %49s %d %9s %lf",
       arr[n].surname,
       arr[n].name,
       &arr[n].year,
       arr[n].gender,
       &arr[n].height)
```

Разбор:

-   `%s` - строка до пробела
-   `%d` - целое число
-   `%lf` - вещественное число типа `double`

Почему строки передаются без `&`, а числа с `&`:

-   строка в C - это массив символов, и имя массива уже указывает на адрес
-   обычная числовая переменная требует передачу адреса

Почему используются ограничения `%49s` и `%9s`:

-   это защита от переполнения массива
-   если в массиве 50 символов, безопасно считывать максимум 49, чтобы оставить место для `\0`

Чтение идёт в цикле:

```while (fscanf(...) == 5)```

Почему `== 5`:

-   ожидается 5 успешно считанных элементов
-   если формат строки нарушен, условие перестанет выполняться

6\. Сравнение строк в языке C
-----------------------------

Одна из ключевых тем этой задачи - строки.

В языке C строку нельзя сравнивать оператором `==`, потому что строка --- это массив символов, а `==` сравнивает адреса, а не содержимое.

Неправильно:

```if (a.name == b.name)```

Правильно:

```strcmp(a.name, b.name)```

Функция `strcmp`:

-   возвращает 0, если строки равны
-   возвращает отрицательное число, если первая строка меньше второй
-   возвращает положительное число, если первая строка больше второй

Это позволяет использовать её для сортировки.

Пример:

```strcmp(a.surname, b.surname)```

означает:\
сравнить две фамилии по алфавиту.

7\. Сортировка по одному полю
-----------------------------

Если бы задача требовала сортировать только по одному полю, логика была бы простой:

-   если сортировка по году → сравнивать `year`
-   если по росту → сравнивать `height`
-   если по имени → использовать `strcmp`

Например:

```if (a.year > b.year)```

или

```if (strcmp(a.name, b.name) > 0)```

Но в этой задаче требуется больше: возможность сортировки по нескольким полям.

* * * * *

8\. Сортировка по нескольким полям
----------------------------------

Смысл сортировки по нескольким полям такой:

Сначала выбирается главное поле.
Например:

-   год рождения

Если у двух записей значения по главному полю совпадают, то используется второе поле.
Например:

-   пол
-   фамилия
-   имя

Пример:

1.  сначала сортировать по году
2.  если годы одинаковые - по фамилии

Именно поэтому в программе используется два параметра сортировки:

```int f1, f2;```

-   `f1` - главное поле
-   `f2` - дополнительное поле

* * * * *

9\. Функция сравнения
---------------------

Чтобы не писать одну и ту же логику сортировки много раз, удобно вынести сравнение в отдельную функцию:

```int compare(struct human a, struct human b, int f1, int f2)```

Эта функция:

-   принимает две записи
-   принимает номера полей сортировки
-   возвращает результат сравнения

Что делает функция:

1.  сравнивает записи по первому полю
2.  если по первому полю элементы различаются - сразу возвращает результат
3.  если по первому полю значения одинаковые - сравнивает по второму

Это делает код:

-   короче
-   понятнее
-   удобнее для расширения

10\. Функция обмена
-------------------

При сортировке элементы массива меняются местами.

Для этого используется функция:

```void swap(struct human *a, struct human *b)```

Почему указатели:

-   нужно изменить реальные элементы массива
-   если передавать копии, обмен не повлияет на исходные данные

Внутри используется временная переменная:

```
struct human temp = *a;\
*a = *b;\
*b = temp;
```

Это стандартный способ обмена двух элементов.

11\. Сортировка методом пузырька
--------------------------------

В задаче используется пузырьковая сортировка.

Принцип:

-   сравниваются соседние элементы массива
-   если порядок неправильный - элементы меняются местами
-   после каждого прохода самый большой элемент "всплывает" в конец

Реализация:

```
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - i - 1; j++) {
        if (compare(arr[j], arr[j + 1], f1, f2) > 0) {
            swap(&arr[j], &arr[j + 1]);
        }
    }
}
```

Почему этот метод подходит:

-   он простой
-   его легко объяснить
-   для учебных задач и небольших массивов он вполне достаточен

12\. Выбор поля через консоль
-----------------------------

Важная часть задачи --- пользователь сам задаёт критерий сортировки.

Например:

-   1 - год
-   2 - имя
-   3 - фамилия
-   4 - пол
-   5 - рост

Это позволяет сделать программу универсальной.

Пример ввода:

```
scanf("%d", &f1);
scanf("%d", &f2);
```

Таким образом, одна и та же программа может сортировать по разным полям без изменения кода.
