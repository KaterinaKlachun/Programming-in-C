1\. Рекурсивные функции
-----------------------

Функция --- основной структурный элемент программы

Рекурсивная функция --- это функция, которая вызывает саму себя.

Пример в задаче:

```
FibonacciSum(index + 1, b, a + b);
```

Особенность:

-   каждый вызов создаёт новый уровень выполнения
-   используется стек вызовов

* * * * *

2\. Последовательность Фибоначчи
--------------------------------

Последовательность строится по правилу:

```
F1 = 1F2 = 1Fn = Fn-1 + Fn-2
```

Пример:

```
1 1 2 3 5 8 ...
```

В программе:

-   `a` --- текущее число
-   `b` --- следующее число

* * * * *

3\. Нелокальный переход
-----------------------

Используются функции:

```
setjmp()longjmp()
```

Принцип:

-   `setjmp()` сохраняет состояние программы
-   `longjmp()` возвращает выполнение в эту точку

Это альтернатива обычному возврату через `return`

* * * * *

4\. Работа со стеком
--------------------

При рекурсии каждый вызов функции помещается в стек.

Стек --- структура данных, где элементы добавляются и удаляются с вершины

При обычной рекурсии:

-   возврат происходит постепенно
-   каждый вызов завершается отдельно

В данной программе:

-   используется `longjmp()`
-   происходит мгновенный выход из всех вызовов

* * * * *

5\. Консольный ввод и вывод
---------------------------

Ввод:

```
scanf("%d", &n);
```

Вывод:

```
printf("%d", a);
```

Согласно лекциям:

-   `%d` --- целое число
-   `&` --- передача адреса переменной

* * * * *

6\. Условный оператор
---------------------

Используется:

```
if (index > n)
```

Назначение:

-   остановка рекурсии
-   переход через `longjmp()`

Условный оператор реализует выбор действия в зависимости от условия

* * * * *

7\. Глобальные переменные
-------------------------

Объявлены:

```
int n;int sum;
```

Согласно лекциям:

-   глобальные переменные доступны во всей программе
-   используются для передачи данных между функциями

8\. Логика вычисления суммы
---------------------------

Алгоритм:

1.  вывести текущее число
2.  прибавить к сумме
3.  перейти к следующему числу

Пример:

```
sum = sum + a;
```

* * * * *

9\. Проверка данных
-------------------

Перед выполнением:

```
if (n <= 0)
```

Назначение:

-   защита от некорректного ввода
-   предотвращение ошибки выполнения

* * * * *

10\. Логика работы программы
----------------------------

1.  Ввести число `n`
2.  Сохранить точку возврата (`setjmp`)
3.  Запустить рекурсивную функцию
4.  Вычислять числа Фибоначчи
5.  Суммировать значения
6.  При достижении конца вызвать `longjmp`
7.  Вернуться в `main()`
8.  Вывести сумму

* * * * *

11\. Итоговый алгоритм
----------------------

1.  Объявить переменные
2.  Ввести `n`
3.  Проверить корректность
4.  Вызвать `setjmp()`
5.  Запустить рекурсию
6.  Вычислять числа
7.  Накопить сумму
8.  Выполнить `longjmp()`
9.  Вывести результат

* * * * *

в задачи **не используется longjmp "по необходимости"**, а демонстрационно.\
Эту задачу можно решить без него (обычным return).

**зачем тут longjmp?**
> Использование `longjmp()` демонстрирует механизм нелокального перехода и позволяет выйти из глубоко вложенной рекурсии без последовательного возврата через стек вызовов.
