1\. Типы данных
---------------

В программе используются типы:

```
longdouble
```

Согласно лекциям:

-   `long` --- целочисленный тип
-   `double` --- вещественный тип

Тип `double` хранится сложнее:

-   знак
-   порядок
-   мантисса

* * * * *

2\. Представление данных в памяти
---------------------------------

Любая переменная в памяти хранится в виде последовательности битов.

Например:

```
1 байт = 8 бит
```

Размер типа определяется через:

```
sizeof(тип)
```

Таким образом:

```
sizeof(long) * 8
```

даёт количество битов числа

* * * * *

3\. Указатели
-------------

Указатель --- это переменная, содержащая адрес в памяти

Пример:

```
unsigned long *p;
```

Оператор:

```
&p
```

возвращает адрес переменной

Разыменование:

```
*p
```

даёт значение по адресу

* * * * *

4\. Приведение типов
--------------------

Используется:

```
p = (unsigned long*)&x;
```

Это означает:

-   берём адрес переменной
-   интерпретируем его как другой тип

Важно:

-   данные не изменяются
-   меняется только способ их чтения

* * * * *

5\. Побитовые операции
----------------------

Используются операции:

```
>>  // сдвиг вправо&   // побитовое И
```

Пример:

```
(*p >> i) & 1
```

Что происходит:

1.  число сдвигается
2.  нужный бит попадает в конец
3.  проверяется значение

Результат:

-   1 --- бит установлен
-   0 --- бит не установлен

Побитовые операции выполняются над каждым битом отдельно

* * * * *

6\. Цикл for
------------

Используется:

```
for (int i = 0; i < sizeof(...) * 8; i++)
```

Цикл `for` применяется для повторения действий заданное число раз

Согласно лекциям:

-   инициализация
-   условие
-   изменение счётчика

* * * * *

7\. Подсчёт единичных битов
---------------------------

Алгоритм:

1.  взять число
2.  проверить каждый бит
3.  если бит = 1 → увеличить счётчик

```
if ((*p >> i) & 1){    count++;}
```

* * * * *

8\. Работа с double
-------------------

Тип `double` занимает 8 байт

Для работы используется:

```
unsigned long long *p;
```

Это связано с тем, что:

-   8 байт = 64 бита
-   требуется тип соответствующего размера

Данные анализируются как последовательность битов

* * * * *

9\. Ввод и вывод данных
-----------------------

Ввод:

```
scanf("%ld", &a);scanf("%lf", &b);
```

Вывод:

```
printf("%d", result);
```

Согласно лекциям:

-   `%ld` --- long
-   `%lf` --- double
-   `%d` --- int

11\. Итоговый алгоритм
----------------------

1.  Объявить переменные
2.  Ввести данные

4.  Привести тип указателя
5.  Пройти по всем битам
6.  Проверить каждый бит
7.  Увеличить счётчик

* * * * *

программа не считает "значение числа" --- а анализирует **его битовое представление**\
Для `double` работаем с **внутренним форматом хранения**, а не с самим числом

**почему для double другой указатель?**

> Потому что тип `double` занимает 8 байт, и для корректного доступа к его битам используется тип `unsigned long long`, соответствующий 64-битному представлению.
